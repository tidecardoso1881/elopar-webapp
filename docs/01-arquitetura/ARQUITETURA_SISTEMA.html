Arquitetura em Camadas do Sistema

O Elopar é um sistema full-stack construído com Next.js 16, integrado ao Supabase para backend. A arquitetura segue um padrão moderno de aplicações serverless com componentes de servidor e cliente.

Fluxos de Dados Principais

Exemplos de fluxos de dados para operações críticas do sistema, mostrando como os dados fluem entre as camadas.

1️⃣ Autenticação e Login
Fluxo: Usuário preenche credenciais → Browser envia para Next.js → Server Action valida com Supabase Auth → JWT retorna → Cliente armazena token → RLS ativado para requisições futuras
Tecnologias: Supabase Auth, JWT, Session Management, RLS Policies
2️⃣ Carregamento do Dashboard
Fluxo: Next.js Server Component renderiza no servidor → Executa query ao PostgreSQL com contexto do usuário → RLS valida acesso → Retorna KPIs agregados → HTML é enviado ao cliente → Componentes interativos hidratam
Tecnologias: Server Components, RLS, PostgreSQL Aggregate Functions
3️⃣ Busca de Profissionais
Fluxo: Usuário digita no campo de busca (Client Component) → onChange dispara Server Action → SQL query com filtros → Resultados retornam com RLS aplicado → UI atualiza via React state → Cards de profissional renderizam
Tecnologias: Server Actions, Full-text Search, Client State Management
4️⃣ Alerta de Renovação
Fluxo: Server Component executa query de datas → Filtra profissionais com renovação próxima → Supabase Realtime notifica mudanças → Componente se revalida → Alert cards aparecem na UI → Usuário clica para ação
Tecnologias: Supabase Realtime, Next.js ISR, Server Components
5️⃣ Submissão de Formulário
Fluxo: Usuário submete form → Client Component valida dados → Server Action persiste no PostgreSQL → Supabase retorna confirmação → UI mostra toast de sucesso → Dados são revalidados no servidor → Cache é limpo
Tecnologias: Server Actions, Form Validation, Revalidation, Toast Notifications

Estrutura de dados do PostgreSQL com as tabelas principais, campos e relacionamentos. O sistema utiliza RLS para controlar acesso aos dados.

Detalhes das Tabelas

📋 Clients (Clientes)

Campo Tipo Descrição
id UUID Chave primária
profile_id UUID FK Referência ao perfil do usuário
name VARCHAR Nome do cliente/empresa
description TEXT Descrição ou observações
status ENUM ativo, inativo, suspenso
kpi_value INTEGER KPI ou valor agregado

👤 Professionals (Profissionais)

Campo Tipo Descrição
id UUID Chave primária
client_id UUID FK Cliente associado
name VARCHAR Nome do profissional
cpf VARCHAR CPF (único)
status ENUM ativo, inativo, férias

🏖️ Vacations (Férias)

Campo Tipo Descrição
id UUID Chave primária
professional_id UUID FK Profissional em férias
start_date DATE Data de início
end_date DATE Data de término
reason VARCHAR Motivo das férias

🔧 Equipment (Equipamentos)

Campo Tipo Descrição
id UUID Chave primária
professional_id UUID FK Profissional proprietário
name VARCHAR Nome do equipamento
type VARCHAR Tipo/categoria
renewal_date DATE Data de renovação/vencimento

Pipeline de Integração e Deploy Contínuo

O Elopar utiliza GitHub Actions para automação de testes, build e deploy. O pipeline garante qualidade de código antes de cada deploy para produção no Vercel.

📊 Pipeline CI/CD Completo
Push para GitHub → GitHub Actions Triggered → Ambiente Setup → Lint & Format
Type Check → Unit Tests → Integration Tests → Build Artifacts
Deploy to Vercel → E2E Tests → Smoke Tests → Production Ready
