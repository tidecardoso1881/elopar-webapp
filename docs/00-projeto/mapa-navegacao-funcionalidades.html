Dashboard de Saúde dos Testes
EP-NEW-003 · ✅ Implementado
Rota atual /area-usuario/saude-testes
Acesso via Card dentro de Área do Usuário
Visível para ⚠️ Semi-oculta
Problema: Só aparece como card dentro de /area-usuario. Usuário precisa saber que deve entrar na Área do Usuário para encontrá-la. Não está na sidebar principal.
💡 Recomendação: Adicionar na sidebar, seção "Admin", visível para roles admin e gerente.
👥
Gerenciamento de Usuários (Admin)
EP-NEW-005A · ✅ Implementado
Rota atual /area-usuario/gerenciar-usuarios
Acesso via Card dentro de Área do Usuário
Visível para 🔴 Enterrada
Problema: Uma das funcionalidades mais críticas para admin está enterrada 2 níveis abaixo (/area-usuario → card → /gerenciar-usuarios). Não aparece na sidebar.
💡 Recomendação: Adicionar diretamente na sidebar, seção "Administração", visível apenas para role admin.
📋
Audit Log
EP-NEW-007 · ✅ Implementado
Rota atual /area-usuario/audit-log
Acesso via Card dentro de Área do Usuário
Visível para 🔴 Enterrada
Problema: Enterrada dentro de Área do Usuário. Para um audit log — ferramenta que requer acesso rápido em incidentes — isso é crítico. Não está na sidebar.
💡 Recomendação: Adicionar na sidebar, seção "Administração", visível para admin e gerente.
🔍
Filtros Avançados com Favoritos
EP-NEW-008 · ✅ Implementado
Rota atual /profissionais (painel embutido)
Acesso via Painel de filtros na lista de profissionais
Visível para 🔵 Contextual
Problema de descoberta: A funcionalidade de salvar filtros favoritos existe dentro do painel de filtros, mas não é óbvia. Muitos usuários usam filtros sem perceber que podem salvá-los com um nome.
💡 Recomendação: Está no lugar correto. Melhorar visibilidade do botão "Salvar filtro" no painel — talvez com tooltip ou destaque na primeira vez.
📊
Dashboard de Métricas
EP-NEW-012 · ✅ Implementado
Rota atual /area-usuario/metricas
Acesso via Sidebar (condicional por e-mail!)
Visível para 🔴 Crítico
Problema CRÍTICO: A sidebar só mostra o link de Métricas se user.email === process.env.METRICS_ALLOWED_EMAIL. Ou seja, apenas 1 usuário específico vê o item na sidebar. Todos os outros precisam conhecer a URL de cor.
💡 Recomendação: Substituir a lógica por email pela verificação de role (admin / gerente). Adicionar como item fixo na sidebar nessa seção.
🕐
Histórico de Alterações por Profissional
EP-NEW-013 · ✅ Implementado
Rota atual /profissionais/[id] (seção na página)
Acesso via Componente ProfessionalHistorico na página de detalhe
Visível para ✅ Correto
✅ Está bem posicionado. O histórico de alterações faz sentido estar na página do profissional. Sugestão: expor como aba dedicada no detalhe em vez de seção rolável.
📝
Notas Internas
EP-NEW-014 · ✅ Implementado
Rota atual /profissionais/[id] (seção na página)
Acesso via Componente ProfessionalNotes — só para admin/gerente via RLS
Visível para 🔵 Contextual
✅ Posicionamento correto. Notas são contextuais ao profissional. O controle por role (admin/gerente) via RLS está adequado. Sugestão: exibir aba "Notas" com contador no cabeçalho para roles autorizados.

Notas sobre a sidebar proposta

  • ● Menu Principal — sem mudanças. Os 7 itens existentes continuam para todos os roles.
  • ● Seção Analytics — nova seção com Dashboard de Métricas e Saúde dos Testes.
    Visível para roles admin e gerente.
    Remove a lógica por e-mail (METRICS_ALLOWED_EMAIL) que só mostrava Métricas para 1 usuário.
  • ● Seção Administração — nova seção com Gerenciar Usuários, Audit Log e Permissões.
    Audit Log visível para admin + gerente.
    Usuários e Permissões: somente admin.
  • ● Área do Usuário — mantida como destino para perfil pessoal (foto, dados), mas deixa de ser o hub escondido das ferramentas admin.

Substituir em layout.tsx:
showMetrics={'{'}user.email === process.env.METRICS_ALLOWED_EMAIL{'}'}
Por verificação de role no perfil do usuário (já disponível no layout via Supabase).

Ações Recomendadas

Ordenadas por impacto na descoberta das funcionalidades

Crítico
🔴 Corrigir lógica de exibição do Dashboard de Métricas
Em src/app/(dashboard)/layout.tsx, substituir showMetrics={'{'}user.email === process.env.METRICS_ALLOWED_EMAIL{'}'} por verificação de role (profile.role === 'admin' || profile.role === 'gerente'). Hoje apenas 1 usuário vê o link.
Alta
🟡 Adicionar seção "Administração" na Sidebar
Criar nova seção na sidebar com links diretos para Gerenciar Usuários, Audit Log e Permissões. Controlar visibilidade por role no componente Sidebar, recebendo o role via prop do layout.
Alta
🟡 Adicionar seção "Analytics" na Sidebar
Criar seção "Analytics" com links para Dashboard de Métricas e Saúde dos Testes, visíveis para admin e gerente. Unificar com a correção do item anterior.
Média
🔵 Transformar Histórico e Notas em abas no detalhe
Refatorar /profissionais/[id]/page.tsx para usar um sistema de abas. Histórico e Notas ganhariam acesso com 1 clique em vez de scroll. Notas mostrariam badge com contagem para admin/gerente.
Média
🔵 Destacar "Salvar Filtro" no painel de profissionais
Adicionar destaque visual ou tooltip "Salve este filtro para usar depois ⭐" na primeira vez que o usuário aplica um filtro. A funcionalidade existe mas não é descobrível.
Baixa
🟢 Revisar Área do Usuário como hub de ferramentas
Após mover as ferramentas admin para a sidebar, a Área do Usuário pode focar apenas em Perfil pessoal e Preferências. Remover os cards de ferramentas admin de lá para evitar duplicidade.
